# Assignment 3 Rubric Checklist

Use this file to track implementation coverage as the project moves from scaffold to final submission.

## High-Fidelity Interface

- [ ] Polished home, landing page, or dashboard
- [ ] Clear navigation structure
- [ ] At least one primary task workflow
- [ ] At least one form, search, or input-based interaction
- [ ] At least one support, help, or informational screen
- [ ] At least one feedback, success, empty, or error state
- [ ] Realistic interface text, labels, and calls to action
- [ ] Consistent visual hierarchy, spacing, typography, and interaction patterns
- [ ] 5-7 connected screens, pages, or views

## Functional User Flows

- [ ] First-time user setup flow
- [ ] Typical course planning flow
- [ ] High-stress alert or registration hold flow
- [ ] Prerequisite recovery flow
- [ ] Each flow includes entry point, user action, system feedback, and completion state

## Accessibility and Responsive Design

- [ ] Readable text contrast
- [ ] Clear heading structure
- [ ] Consistent navigation labels
- [ ] Keyboard-friendly interaction
- [ ] Meaningful button and form labels
- [ ] Clear error messages
- [ ] Color is not the only way meaning is communicated
- [ ] Desktop layout
- [ ] Tablet layout
- [ ] Mobile layout

## Deployment and Submission

- [ ] Vercel deployment loads without instructor setup
- [ ] Live site matches submitted codebase
- [ ] README includes setup/run/deployment information
- [ ] Source code is submitted as ZIP or repository link
- [ ] Final report PDF is complete
- [ ] Screenshots are included in report appendix
